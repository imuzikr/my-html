GitHub Actions로 HTML 아티클 자동 배포하기

HTML 파일을 articles/ 폴더에 넣고 main 브랜치에 머지하면, GitHub Actions가 자동으로 목차 페이지를 생성하고 GitHub Pages에 배포하는 구조를 설명한다. 각 파일이 어떤 역할을 하는지, 글이 발행되기까지 어떤 단계를 거치는지를 다룬다.

01

전체 그림 먼저 보기

이 프로젝트는 HTML로 작성한 글(아티클)을 웹에 자동으로 발행하는 시스템이다. 글을 정해진 폴더에 넣기만 하면, 나머지는 자동으로 처리된다.

목차 페이지가 만들어지고, 인터넷에 공개되는 것까지 사람이 추가로 할 일이 없다. 블로그 플랫폼(네이버 블로그, 티스토리 등)을 직접 만든 것과 비슷한데, 외부 서비스에 의존하지 않고 GitHub라는 개발자 도구 위에서 모든 것을 운영한다는 차이가 있다.

글 작성 → main 머지 → Actions 자동 실행 → 웹 배포까지의 전체 흐름

02

프로젝트 폴더 구조

프로젝트는 아래와 같은 구조로 이루어져 있다. 각 파일과 폴더는 독립적인 역할을 맡고 있다.

프로젝트 구조
my-html/
├── .claude/                        ← Claude Code 설정
│   └── commands/html.md            ← /html 커맨드 작성 규칙
├── .github/
│   └── workflows/
│       └── deploy.yml              ← 자동 배포 규칙 (GitHub Actions)
├── scripts/
│   └── generate_index.py           ← 목차 페이지 생성 스크립트
├── articles/
│   └── 2026-05-19-ontology.html    ← 아티클 (글)
├── design-system.html              ← 디자인 규칙 참고 파일
├── html-skill-for-claude-ai.md     ← Claude 글 작성 지시문
└── index.html                      ← 자동 생성되는 목차 페이지

index.html은 자동 생성 파일이다

목차 파일인 index.html은 직접 편집하는 파일이 아니다. generate_index.py 스크립트가 매번 새로 만들어낸다. 직접 수정해도 다음 배포 시점에 덮어씌워진다.

03

각 파일이 하는 일

.github/workflows/deploy.yml

deploy.yml — 자동화의 두뇌

GitHub Actions에게 "언제, 무엇을, 어떻게 실행할지" 알려주는 설계도다. 이 파일이 .github/workflows/ 폴더 안에 존재하는 것만으로 자동화가 등록된다. 별도로 활성화 버튼을 누를 필요가 없다.

항목내용
언제?main 브랜치에 코드가 푸시되면
어디서?GitHub가 제공하는 클라우드 서버(Ubuntu)에서
무엇을?Python 스크립트를 실행하고, 결과를 웹에 배포
수동 실행?Actions 탭 → "Run workflow" 버튼으로 언제든 가능
scripts/generate_index.py

generate_index.py — 목차 자동 생성기

articles/ 폴더를 스캔해서 안에 있는 HTML 파일 목록을 모두 수집한 뒤, 각 글로 연결되는 링크 카드가 포함된 index.html을 자동으로 만들어낸다.

핵심 포인트

기존 index.html에 링크를 하나 추가하는 것이 아니라 매번 처음부터 새로 생성한다. 글을 추가하든, 삭제하든, 파일명을 바꾸든 항상 그 시점의 최신 상태가 반영된다.

articles/

articles/ 폴더 — 글이 저장되는 곳

실제 아티클 HTML 파일이 들어가는 폴더다. 새 글을 발행하고 싶으면 이 폴더에 HTML 파일을 넣으면 된다. 파일명은 2026-05-18-title.html처럼 날짜와 주제를 조합해서 짓는다.

design-system.html

design-system.html — 디자인 규칙서

아티클의 색상, 폰트, 간격, 카드 모양 같은 시각 디자인 규칙이 정의된 참고 파일이다. 새 글을 만들 때 이 파일을 기준으로 삼으면 모든 아티클의 디자인이 통일된다.

html-skill-for-claude-ai.md

html-skill-for-claude-ai.md — Claude를 위한 작성 지시문

Claude에게 HTML 아티클을 만들어달라고 요청할 때 따르는 작성 규칙서다. /html article ○○○처럼 명령하면, Claude가 이 규칙에 맞춰 디자인 시스템을 적용한 완성도 높은 HTML을 만들어준다.

두 파일의 관계

design-system.html은 "어떤 모양으로 만들지"를 정의하고, html-skill-for-claude-ai.md는 "어떤 내용 구성으로 만들지"를 정의한다. 잡지로 비유하면, 하나는 아트디렉터의 디자인 매뉴얼이고, 다른 하나는 편집장의 기사 작성 가이드다.

04

새 글이 발행되는 전체 과정

1

글 작성

Claude에게 /html article ○○○라고 요청하면 디자인 규칙에 맞는 HTML 파일이 만들어진다. 또는 직접 HTML을 작성해도 된다.

2

articles/ 폴더에 파일 추가

만들어진 HTML 파일을 articles/ 폴더에 넣고 커밋한다. 이 시점에서는 아직 배포되지 않는다.

3

main 브랜치에 푸시

git push origin HEAD:main으로 main에 직접 반영한다. 이 순간 GitHub Actions가 트리거된다.

4

GitHub Actions 자동 실행

deploy.yml이 실행되어 generate_index.py가 새 아티클을 포함한 index.html을 재생성한다.

5

GitHub Pages 배포 완료

생성된 파일 전체가 GitHub Pages에 업로드되고, 약 1~2분 후 웹에서 확인할 수 있다.

3단계부터는 사람이 할 일이 없다

main에 푸시하는 순간부터 목차 재생성 → 배포까지 전부 자동이다. 평균 1~2분이면 웹에서 새 글을 확인할 수 있다.

05

배포를 위해 필요한 GitHub 설정

자동 배포가 처음 작동하려면 GitHub 레포지토리에서 두 가지 설정을 해두어야 한다.

이 설정이 빠지면 배포가 실패한다

아래 두 설정은 레포지토리를 처음 만들 때 한 번만 하면 된다. 이후에는 건드릴 필요가 없다.

설정 위치변경 값이유
Settings → Pages → Source GitHub Actions 선택 GitHub에게 "배포는 워크플로가 처리한다"고 알려줌
Settings → Environments → github-pages main 브랜치 배포 허용 이 설정이 없으면 Branch "main" is not allowed to deploy 에러 발생

핵심 정리

한 문장 요약

글을 articles/ 폴더에 넣고 main에 머지하면, 나머지는 GitHub Actions가 알아서 목차를 만들고 웹에 배포해준다.
