Web Dev Basics

웹 개발 입문,
Next.js · Vite · SPA · 라우팅

라우트는 메뉴, 라우팅은 주문 처리, 라우터는 직원. 식당 비유 하나로 현대 웹 개발의 핵심 개념을 끝까지 따라갈 수 있습니다.

프론트엔드 백엔드 SPA 라우팅 웹 입문
핵심 요약

웹 앱은 화면(프론트엔드)과 뒤편 일꾼(백엔드)으로 나뉩니다. Next.js는 백엔드가 있게도 없게도 만들 수 있는 유연한 도구이고, Vite는 기본적으로 화면을 만드는 도구입니다. 현대 웹 앱의 다수는 SPA 방식 — 한 장의 HTML을 빈 그릇으로 받아 자바스크립트로 내용을 채우고, 페이지를 옮길 때도 라우터가 그 그릇 위에서 내용만 갈아끼웁니다.

1

먼저 알아둘 두 가지: 프론트엔드와 백엔드

웹 앱은 크게 두 부분으로 나뉩니다.

🖥️
프론트엔드 (Frontend)
사용자가 눈으로 보고 클릭하는 화면입니다. 버튼, 글자, 이미지처럼 브라우저에 그려지는 모든 것이죠.
⚙️
백엔드 (Backend)
사용자 눈에 보이지 않는 뒤편의 일꾼입니다. 데이터 저장, 로그인 처리, 계산 등을 서버(항상 켜져 있는 컴퓨터)에서 담당합니다.
"서버 코드 = 백엔드 코드"라고 이해하시면 됩니다.
2

라우트 · 라우팅 · 라우터 — 식당 비유로 한 번에

이 세 단어는 모양이 비슷해서 헷갈리지만, 사실 하나의 이야기에서 나온 가족 같은 개념입니다. 식당에 비유해 보겠습니다.

여러분이 식당에 가면 메뉴판을 보고 "비빔밥 주세요"라고 주문하죠. 그러면 직원이 주문을 듣고, 주방에 전달해서, 비빔밥을 가져다줍니다. 웹사이트에서도 똑같은 일이 일어납니다.

손님이 주문(주소 입력)하면 직원(라우터)이 주문 처리(라우팅)를 해서 알맞은 음식(화면)을 내줍니다

🍚 라우트(Route) = 메뉴판의 각 메뉴

웹사이트 주소(URL)의 각 경로 하나하나를 말합니다. /home(홈), /about(회사 소개), /products(상품 목록)처럼, 주소창에 들어가는 각각의 길이 바로 '라우트'입니다. 메뉴판에 적힌 '비빔밥', '김치찌개' 같은 메뉴 하나하나라고 보시면 됩니다.

📖
용어

라우트는 우리말로 '경로'라는 뜻입니다.

🛎️ 라우팅(Routing) = 주문을 받아 알맞은 음식을 내주는 과정

사용자가 특정 주소로 들어왔을 때, "이 주소에는 이 화면을 보여줘야지" 하고 알맞은 화면을 연결해주는 작업입니다. "/about 주소로 들어왔으니 회사 소개 화면을 보여준다" — 이 판단과 연결의 과정 전체가 '라우팅'입니다. 직원이 주문을 듣고 알맞은 음식을 내주는 행위에 해당합니다.

📖
용어

라우팅은 '경로를 안내하는 일' 정도로 이해하면 됩니다.

🧑‍🍳 라우터(Router) = 그 일을 담당하는 직원

라우팅이라는 일을 실제로 처리해주는 담당자(프로그램)입니다. 주소를 보고 어떤 화면을 보여줄지 정해주는 똑똑한 직원이라고 생각하면 됩니다. 개발자는 이 라우터에게 "/products 주소로 오면 상품 목록 화면을 보여줘"라고 미리 규칙을 알려줍니다.

한 줄 정리: 라우트(메뉴) 여러 개를 두고, 사용자가 주소를 입력하면, 라우터(직원)가 라우팅(주문 처리)을 해서 알맞은 화면을 보여줍니다.
용어식당 비유실제 의미
라우트 (Route) 메뉴판의 메뉴 하나 주소(URL)의 경로 하나 (/about 등)
라우팅 (Routing) 주문 받아 음식 내주는 과정 주소에 맞는 화면을 연결하는 작업
라우터 (Router) 그 일을 하는 직원 라우팅을 처리하는 프로그램

이 개념은 뒤에 나올 SPA 설명에서 다시 등장하니, 가볍게 기억해두시면 됩니다.

3

Next.js — "백엔드가 있을 수도, 없을 수도"

흔히 "Next.js로 앱을 만들었다"고 하면 백엔드가 당연히 있다고 생각하기 쉽지만, 꼭 그렇지는 않습니다. Next.js는 만드는 방식에 따라 결과물이 달라지는 유연한 도구이기 때문입니다.

📄
백엔드 없이 만들기
완성된 화면 파일(HTML 등)만 뽑아내서 올려두면, 서버에서 돌아가는 코드 없이도 동작합니다.
🔧
백엔드 포함해서 만들기
로그인 처리, 데이터 저장, 화면을 서버에서 미리 그려 보내주기 같은 기능을 쓰면 서버 코드가 함께 동작합니다.
"Next.js 앱"이라는 말만으로는 백엔드 유무를 알 수 없고, 어떻게 만들었는지를 봐야 합니다.
4

Vite — "기본은 화면 만드는 도구"

Vite(비트)는 기본적으로 프론트엔드(화면)를 만드는 도구입니다. 개발을 빠르고 편하게 해주는 역할이 핵심이라, 그 자체만으로는 백엔드를 책임지지 않습니다.

그렇다면 Vite로는 풀스택(프론트엔드 + 백엔드 전부) 개발이 불가능할까요? 그렇지 않습니다. 두 가지 방법이 있습니다.

🔌
백엔드를 따로 두기
Vite로 화면을 만들고, 백엔드는 별도의 프로그램으로 만들어 연결합니다.
🚀
Vite 기반 확장 도구 쓰기
Vite를 엔진처럼 쓰면서 백엔드 기능까지 더해주는 프레임워크(예: Remix, SvelteKit, Nuxt)를 사용하면 풀스택 개발이 됩니다.
📖
용어

풀스택(Full-stack)이란 프론트엔드와 백엔드를 모두 아우르는 개발을 뜻합니다.

5

SPA — "한 장의 페이지로 모든 걸 해결하는 방식"

여기서부터가 핵심입니다. 많은 현대 웹 앱이 SPA(Single Page Application, 단일 페이지 애플리케이션) 방식으로 동작합니다.

전통적인 웹사이트 vs SPA

옛날 방식의 웹사이트(MPA)는 메뉴를 누를 때마다 서버에서 새로운 페이지(HTML) 전체를 통째로 받아왔습니다. 그래서 화면이 한 번 깜빡이며 새로 그려졌죠. 반면 SPA는 단 한 장의 HTML 페이지만 사용합니다. 페이지를 옮겨다니는 것처럼 보여도, 실제로는 같은 페이지 위에서 내용물만 갈아끼우는 것입니다.

MPA는 페이지마다 HTML 전체를 새로 받지만, SPA는 한 장 위에서 내용만 교체합니다

SPA가 처음 화면을 그리는 순서

  1. 빈 페이지를 받습니다. 브라우저가 받아온 HTML은 거의 텅 비어 있습니다. 빈 그릇과, "이 그릇을 채울 코드 파일은 여기 있어"라는 안내만 들어 있죠.
  2. 코드(자바스크립트)를 내려받습니다. 브라우저가 안내를 따라 화면을 그릴 자바스크립트 파일을 가져옵니다.
  3. 화면을 채웁니다. 이 코드가 실행되면서 빈 그릇 안에 실제 버튼, 글자, 이미지가 그려집니다. 이 순간 비로소 사용자에게 화면이 보입니다.
🥣 빈 HTML 도착
빈 그릇 + "코드는 여기 있어" 안내
↓
📥 자바스크립트 다운로드
화면을 그릴 코드 파일을 가져옴
↓
🎨 화면 채우기
버튼·글자·이미지가 그려지고 사용자에게 보임
📖
용어

자바스크립트(JavaScript)는 웹 화면을 움직이게 하고 내용을 채우는 프로그래밍 언어입니다.

다른 페이지로 이동할 때 — 여기서 라우터가 일합니다

SPA에서 다른 메뉴로 이동하면 어떻게 될까요? HTML을 새로 받지 않습니다. 처음 받은 한 장의 HTML을 계속 쓰면서, 그 안의 내용만 자바스크립트가 새 화면으로 교체합니다.

바로 이때 앞에서 배운 라우터(직원)가 활약합니다. 사용자가 주소를 바꾸거나 메뉴를 클릭하면, 라우터가 그 주소(라우트)에 맞는 화면을 골라서 보여주는 라우팅을 합니다. 서버에 새 페이지를 요청하는 게 아니라, 브라우저 안에서 자바스크립트 라우터가 알아서 화면을 바꿔주는 것이죠. 그래서 이걸 '클라이언트 사이드 라우팅'(브라우저 쪽에서 일어나는 라우팅)이라고 부릅니다.

화면을 바꿀 때는 두 가지 경우가 있습니다.

⚡
코드를 미리 다 받아뒀다면
추가 다운로드 없이 즉시 화면이 바뀝니다.
🧩
코드를 페이지별로 나눠뒀다면
그 페이지로 처음 갈 때 필요한 코드 조각만 그제서야 내려받습니다. 한 번 받으면 저장해두고 재사용합니다.

여기에 더해, 화면에 표시할 데이터(예: 상품 목록, 게시글 내용)는 백엔드에 따로 요청해서 받아오기도 합니다.

페이지를 옮길 때 오가는 것은 'HTML 전체'가 아니라 '필요한 코드 조각'과 '데이터'뿐입니다. 그래서 SPA는 화면 전환이 빠르고 부드럽습니다.
6

SPA의 장점과 한계

구분내용
장점 페이지 전환이 깜빡임 없이 빠르고 부드럽습니다. 앱처럼 매끄러운 사용 경험을 줍니다.
단점 1 (검색 노출) 처음 받는 HTML이 비어 있어서, 검색 엔진(구글 등)이 내용을 제대로 읽지 못할 수 있습니다(SEO 문제).
단점 2 (첫 화면 지연) 코드를 받아 실행하기 전까지 잠깐 빈 화면이 보일 수 있습니다.

이런 단점을 보완하려고 나온 것이 서버가 화면을 미리 채워서 보내주는 방식입니다. 전문 용어로 SSR(서버 사이드 렌더링)이나 SSG(정적 생성)라고 부르며, 앞서 이야기한 Next.js 같은 도구가 바로 이런 기능을 제공합니다.

한눈에 보는 요약

  • 웹 앱은 화면(프론트엔드)과 뒤편 일꾼(백엔드)으로 나뉩니다.
  • 라우트(주소 경로)는 메뉴, 라우팅(주소에 맞는 화면 연결)은 주문 처리, 라우터(그 일을 하는 프로그램)는 직원에 비유할 수 있습니다.
  • Next.js는 백엔드가 있게도, 없게도 만들 수 있는 유연한 도구입니다.
  • Vite는 기본적으로 화면 만드는 도구지만, 다른 도구와 결합하면 풀스택 개발도 가능합니다.
  • SPA는 한 장의 HTML을 빈 그릇으로 받은 뒤, 자바스크립트로 내용을 채우고, 페이지를 옮길 때도 라우터가 그 그릇 위에서 내용만 바꾸는 방식입니다.
  • 그래서 SPA는 빠르고 부드럽지만, 검색 노출과 첫 화면 지연이라는 약점이 있고, 이를 보완하는 기술(SSR, SSG 등)이 계속 발전하고 있습니다.

메뉴판(라우트)과 직원(라우터)만 기억하면, 현대 웹의 절반은 이해한 셈입니다.

웹 개발을 처음 접하는 분들을 위해 핵심 개념을 비유 중심으로 정리한 글입니다.
