LLM은 어떻게 말을 이해하고 만들어 낼까?

고등학생을 위한 LLM 작동 원리, 프롬프트 엔지니어링, 그리고 책임 있는 AI 사용 수업

토큰 벡터 어텐션 다음 토큰 예측 책임 있는 사용
Big Picture

오늘의 핵심 문장

LLM은 사람의 언어를 그대로 이해하는 것이 아니라, 언어를 숫자와 벡터로 바꾸고, 여러 신경망 층에서 계산한 뒤, 다음 토큰을 예측하며 문장을 만들어 내는 모델입니다.

수업의 목표는 “AI가 똑똑하다”가 아니라 “AI가 어떻게 계산하고, 우리는 어떻게 판단해야 하는가”입니다.

Roadmap

언어가 답변이 되기까지

1. 쪼개기

문장을 토큰으로 나누고 번호를 붙입니다.

2. 표현하기

번호를 의미 좌표인 임베딩 벡터로 바꿉니다.

3. 계산하기

어텐션과 여러 층이 문맥 관계를 계산합니다.

4. 배우기

예측, 비교, 수정을 반복하며 가중치를 조정합니다.

5. 생성하기

다음 토큰 예측을 반복해 답변을 만듭니다.

6. 판단하기

그럴듯한 답도 검토하고 사람이 책임집니다.

Token

컴퓨터는 문장을 작은 조각으로 나눈다

사람

단어를 보면 의미, 경험, 상황을 떠올립니다.

컴퓨터

문자를 데이터로 저장하고 계산 가능한 형태로 바꿉니다.

토큰은 문장을 처리하기 위해 나눈 작은 조각입니다. “나는 학교에 간다”는 “나는 / 학교 / 에 / 간다”처럼 나뉠 수 있습니다.

ID vs Meaning

번호만으로는 의미의 가까움을 알 수 없다

고양이 = 10

번호는 단지 출석번호입니다.

자동차 = 11

번호가 가깝다고 의미가 가까운 것은 아닙니다.

강아지 = 500

의미는 가깝지만 번호는 멀 수 있습니다.

그래서 LLM은 토큰 번호를 의미를 계산할 수 있는 벡터로 바꿉니다.

Embedding

벡터는 의미를 나타내는 좌표다

벡터는 숫자 하나가 아니라 여러 숫자의 묶음입니다. 단어의 특징을 여러 방향에서 표현합니다.

임베딩은 처음부터 정답 좌표가 아니라, 학습을 통해 계속 조정되는 표현입니다.

Hidden Layers

신경망 층은 벡터를 계속 가공한다

입력 벡터는 그대로 지나가지 않습니다. 여러 층에서 가중치를 곱하고 더하며 중요한 특징을 더 강하게 반영합니다.

Context

같은 단어도 문맥에 따라 달라진다

배를 먹었다

여기서 배는 과일입니다.

배를 탔다

여기서 배는 교통수단입니다.

배가 아프다

여기서 배는 몸의 일부입니다.

문맥을 반영하려면 단어 하나만 보는 것이 아니라 문장 안의 다른 단어들과의 관계를 계산해야 합니다.

Attention

어텐션은 중요한 단어에 형광펜을 치는 과정이다

Attention Detail

어텐션을 조금 더 정확히 말하면

질문 카드

나는 어떤 정보를 찾고 있는가?

이름표 카드

나는 어떤 정보와 관련 있는가?

내용 카드

내가 전달할 정보는 무엇인가?

트랜스포머는 단순히 원래 벡터가 가까운지만 보는 것이 아니라, 각 토큰을 여러 관점으로 바꾼 뒤 관련성을 계산합니다.

Transformer

트랜스포머가 특별한 이유

이전 방식

문장을 앞에서부터 하나씩 읽어야 해서 느리고, 멀리 떨어진 단어 관계를 놓치기 쉬웠습니다.

트랜스포머

문장 전체를 한꺼번에 보며 모든 단어 사이의 관계를 계산합니다. 그래서 먼 거리의 관계도 더 잘 포착합니다.

어텐션은 문장 속 토큰들이 서로를 참고하도록 만드는 핵심 장치입니다.

Training

모델은 예측하고, 비교하고, 수정하며 배운다

  1. 벡터를 입력한다.
  2. 모델이 다음 토큰을 예측한다.
  3. 정답과 비교한다.
  4. 틀린 정도, 즉 오차를 계산한다.
  5. 오차를 줄이는 방향으로 가중치를 수정한다.

이 과정을 수많은 문장에 대해 반복하는 것이 LLM 학습의 기본 구조입니다.

Training vs Use

학습할 때와 사용할 때는 다르다

학습할 때

예측, 비교, 오차 계산을 통해 모델 안의 가중치를 수정합니다.

사용할 때

이미 학습된 가중치를 이용해 현재 대화 문맥에서 다음 토큰을 예측합니다.

일반적인 대화 중에는 모델의 기본 가중치가 매번 즉시 바뀌지는 않습니다. 다만 현재 대화 내용은 문맥으로 참고합니다.

Probability

LLM은 다음 말을 확률로 고른다

Generation

전체 답을 한 번에 꺼내지 않는다

인공지능은

인공지능은 사람의

인공지능은 사람의 언어를

인공지능은 사람의 언어를 처리하는

예측한 토큰을 다시 입력에 포함하고, 다음 토큰을 또 예측합니다. 이 반복이 문장이 됩니다.

Limits

LLM은 사람처럼 경험하지 않는다

잘하는 것

언어 패턴, 문맥 관계, 글의 흐름을 매우 잘 계산합니다.

주의할 것

직접 경험하거나 느끼지는 않으며, 그럴듯하지만 틀린 답을 할 수 있습니다.

LLM의 “이해”는 인간처럼 경험을 떠올리는 이해라기보다, 단어와 문맥의 관계를 계산하는 방식의 이해에 가깝습니다.

Hallucination

왜 그럴듯한데 틀린 말을 할까?

  • LLM은 데이터베이스처럼 정답을 찾아 꺼내는 방식이 아닙니다.
  • 지금 문맥에서 다음에 올 법한 말을 예측하는 방식입니다.
  • 그래서 모르는 내용도 문장 패턴에 맞춰 자연스럽게 이어 말할 수 있습니다.
  • 출처, 날짜, 숫자, 인물 이름, 최신 정보에서 특히 조심해야 합니다.

말이 자연스럽다고 해서 항상 사실이라는 뜻은 아닙니다.

Context Window

LLM은 무한히 기억하지 않는다

책 전체를 항상 보는 것이 아니라, 책상 위에 펼쳐진 몇 쪽을 보며 답하는 것과 비슷합니다.

Prompting

좋은 질문이 좋은 답변을 만든다

Prompt Example

학생용 프롬프트 공식

발표 준비용 프롬프트
너는 고등학생에게 설명하는 과학 선생님이야. 나는 LLM의 작동 원리를 발표하려고 해. 토큰, 임베딩, 어텐션, 다음 토큰 예측을 쉬운 비유와 함께 설명해 줘. 어려운 수식은 빼고, 발표용 5문장으로 정리해 줘.

프롬프트 엔지니어링은 요령이 아니라, 목적과 조건을 분명히 하는 사고 훈련입니다.

Workflow

한 번에 완벽한 답을 기대하지 말자

  1. 초안 만들기
  2. 부족한 점 물어보기
  3. 반례나 오류 찾기
  4. 더 쉽게 바꾸기
  5. 최종 판단은 사람이 하기

LLM은 정답 자판기가 아니라 생각을 발전시키는 도구로 쓰는 편이 좋습니다.

Check

그대로 믿으면 안 되는 항목

숫자 날짜 인물 이름 논문과 기사 출처 법률 의학 진학과 진로
검토용 질문
이 답변에서 사실 확인이 필요한 부분을 따로 표시해 줘. 확실한 내용과 추정한 내용을 나누어 설명해 줘.
Responsibility

최종 판단과 책임은 인간에게 있다

LLM은 강력한 도구이지만, 최종 결과를 이해하고 선택하고 책임지는 주체는 인간이다. AI가 만든 답변을 그대로 제출하거나 믿는 것이 아니라, 사람이 목적에 맞게 검토하고 수정하고 판단해야 한다.
Closing Question
