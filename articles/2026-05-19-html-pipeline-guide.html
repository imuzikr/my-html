HTML 자동 발행 파이프라인
스킬과 GitHub Actions 완전 가이드

/html 스킬로 아티클을 생성하고, GitHub Actions로 자동 배포하는 파이프라인의 전체 구조를 설명한다. 각 구성 요소의 역할, 사용 방법, 새 레포지토리에 설치하는 절차, 다른 AI 서비스에 이식하는 방법까지 한 곳에 정리한 참조 문서다.

01

시스템 개요

이 파이프라인은 AI가 HTML 아티클을 작성하면 자동으로 웹에 발행되는 시스템이다. 글쓴이가 신경 써야 할 일은 주제를 정하고 결과를 확인하는 것뿐이다.

전체 발행 흐름 — 주제 입력부터 웹 공개까지

시스템을 구성하는 두 축

/html 스킬은 AI가 아티클을 만드는 방법을 정의하고, GitHub Actions 파이프라인은 만들어진 파일을 자동으로 배포한다. 두 축이 맞물려 "작성 → 검토 → 발행"의 전 과정을 처리한다.

03

GitHub Actions 파이프라인

main 브랜치에 푸시되는 순간 자동으로 실행되는 워크플로다. 사람이 추가로 할 일은 없다.

deploy.yml — 워크플로 설계도

.github/workflows/deploy.yml
name: Build & Deploy to GitHub Pages

on:
  push:
    branches: ["main"]   # main 푸시 시에만 실행
  workflow_dispatch:      # Actions 탭에서 수동 실행도 가능

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Generate index.html
        run: python3 scripts/generate_index.py

      - uses: actions/configure-pages@v5
      - uses: actions/upload-pages-artifact@v3
        with:
          path: "."
      - uses: actions/deploy-pages@v4

generate_index.py — 목차 생성기

워크플로 안에서 실행되는 Python 스크립트다. 아티클이 추가·삭제될 때마다 index.html을 처음부터 새로 만든다.

기능동작 방식
파일 스캔모든 폴더의 .html 파일을 재귀 탐색 (index.html, design-system.html 제외)
메타데이터 추출<title>, <meta name="description">, <meta property="og:image">, 파일명 날짜
썸네일 처리og:image 있으면 이미지, 없으면 폴더별 그라디언트 + 제목 텍스트
버튼 주입각 아티클에 "목록으로 돌아가기" 버튼을 자동 삽입 (멱등성 보장)
색상 변화같은 폴더 내 카드도 4가지 그라디언트 변형을 순환

index.html은 직접 편집하지 않는다

generate_index.py가 매 배포 시 덮어쓴다. 목차 디자인을 바꾸려면 스크립트 안의 build() 함수를 수정한다.

05

새 레포지토리에 설치하기

1

파일 배치

아래 4개 파일을 레포지토리에 복사한다.

파일배치 경로
deploy.yml.github/workflows/deploy.yml
generate_index.pyscripts/generate_index.py
design-system.htmldesign-system.html (루트)
html.md.claude/commands/html.md
2

GitHub Pages 소스 설정

레포지토리 → Settings → Pages → Source를 GitHub Actions로 변경한다.

3

배포 환경 브랜치 허용

Settings → Environments → github-pages에서 main 브랜치의 배포를 허용한다. 없으면 "Branch is not allowed to deploy" 오류가 발생한다.

4

첫 아티클 작성 및 배포

/html article 첫 번째 글 주제를 실행하고 푸시하면 자동 배포가 완료된다.

2단계와 3단계는 레포당 한 번만

GitHub Pages 설정은 레포지토리를 처음 구성할 때만 필요하다. 이후 아티클을 추가할 때는 파일을 만들고 main에 푸시하기만 하면 된다.

06

다른 AI 서비스에 이식하기

GitHub Actions 파이프라인은 AI와 무관하게 작동한다. 어떤 도구로 HTML을 만들어도 articles/에 넣고 main에 푸시하면 배포된다.

파일이식성다른 AI에서 사용하는 방법
deploy.yml✅ 완전 이식 가능그대로 사용
generate_index.py✅ 완전 이식 가능그대로 사용
design-system.html✅ 완전 이식 가능디자인 참조로 첨부
html.md⚙️ 변환 필요내용을 시스템 프롬프트로 붙여넣기

서비스별 적용 방법

서비스html.md 적용 방법
ChatGPT Custom GPTConfigure → Instructions에 html.md 내용 붙여넣기
Cursor / Windsurf.cursorrules 파일에 작성 규칙 추가
Gemini / 기타 챗봇대화 첫 메시지에 html.md를 시스템 프롬프트로 제공
직접 작성design-system.html의 CSS 변수를 복사해 동일한 디자인으로 작성

AI가 바뀌어도 파이프라인은 그대로

어떤 AI 도구로 HTML을 만들든, articles/에 넣고 main에 푸시하는 행위만으로 목차 재생성과 배포가 자동 실행된다. AI를 바꿔도 GitHub Actions 설정을 다시 건드릴 필요가 없다.

요약

한 문장 요약

/html 스킬로 아티클을 만들고, 검토 후 main에 푸시하면 GitHub Actions가 목차를 재생성하고 GitHub Pages에 자동 배포한다. 파이프라인은 AI 도구에 종속되지 않아 어떤 서비스로도 이식 가능하다.

구성 요소역할핵심 파일
