Code Blocks

Component

async function fetchUserProfile(userId: string): Promise<User> {
  // Check cache first to avoid redundant DB hits
  const cached = await cache.get(`user:${userId}`);
  if (cached) return cached;

  const user = await db.users.findUnique({
    where: { id: userId },
    include: { profile: true, roles: true }
  });

  if (!user) throw new NotFoundError(`User ${userId} not found`);

  await cache.set(`user:${userId}`, user, { ttl: 300 });
  return user;
}

Reusable Patterns

Article

난도 있는 기술 아티클(Firestore 보안 설계 등)에서 검증된 세 가지 설명 패턴입니다. 복사해서 그대로 쓰세요. 모든 색은 디자인 토큰을 사용합니다.

1 · Inline SVG Diagram

외부 의존성 없이 저장소에 그대로 배포되는 자체 완결형 다이어그램. mermaid.ink 같은 외부 렌더러 대신 <svg viewBox>를 직접 작성합니다. 막대·타임라인·시퀀스·계층 등 어떤 형태든 가능하며, 색은 토큰(#3a6b4a green / #2d5a8e blue / #d95f2b accent / #a8322a red)을 씁니다. 화살표는 <marker>로 정의합니다.

코드 = 문서 ID. get은 콕 집어 읽기(허용), list는 전체 훑기(차단).

2 · Bad / Good Compare Cards

안티패턴(✗ 빨강)과 해법(✓ 초록)을 나란히 배치해 대비를 만듭니다. "왜 이게 문제이고 무엇이 답인가"를 한눈에 보여줄 때 효과적입니다.

✗ 코드를 필드에 두면

코드로 찾으려면 전체 훑기(list)가 필요 → 다른 반 코드까지 다 보임.

✓ 코드를 문서 ID로 두면

아는 사람만 get으로 하나에 도달. list는 관리자만 → 우회 차단.

3 · Step List + Summary Pull-quote

번호가 자동 매겨지는 단계 리스트로 전체 절차를 압축하고, 마지막에 한 줄 풀인용으로 핵심을 못 박습니다. 보통 글의 마지막 "한 장 요약" 섹션에서 함께 씁니다.

  1. 코드는 문서 ID로. get은 누구나, list는 교사만.
  2. 만료는 expiresAt으로. 판정은 expiresAt < now 한 번.
  3. 코드 만료 ≠ 추방. 소속은 uid 기준 멤버십에 별도 저장.
코드는 14일짜리 입장권, 멤버십은 평생 정기권. 만료는 새 손님만 막고, 기존 학생은 uid로 어느 기기에서든 다시 들어온다.
HTML · 세 패턴 마크업 
<!-- 1. Inline SVG diagram -->
<div class="diagram-wrap">
  <svg viewBox="0 0 800 230" role="img" aria-label="...">
    <!-- 색: #3a6b4a green / #2d5a8e blue / #d95f2b accent / #a8322a red -->
    <rect .../> <text ...>...</text>
    <path d="M.. L.." marker-end="url(#arrow)"/>
    <defs><marker id="arrow" ...><path d="M0 0 L6 3 L0 6 Z"/></marker></defs>
  </svg>
  <p class="diagram-caption">캡션</p>
</div>

<!-- 2. Bad / Good compare cards -->
<div class="compare-grid">
  <div class="compare-card bad"><h4>✗ 안티패턴</h4><p>...</p></div>
  <div class="compare-card good"><h4>✓ 해법</h4><p>...</p></div>
</div>

<!-- 3. Step list + summary pull-quote -->
<ol class="step-list">
  <li><strong>단계 제목.</strong> 설명</li>
</ol>
<div class="pull-quote">한 줄 핵심 요약</div>
ℹ
CSS는 각 아티클에 인라인
이 패턴들의 클래스(.diagram-wrap, .compare-card, .step-list, .pull-quote)는 공용 스타일시트가 아니라 각 아티클의 <style>에 인라인으로 둡니다. 아티클은 단일 HTML 파일로 독립 배포되기 때문입니다. 위 데모는 .rp- 접두사로 스코프했지만, 실제 아티클에서는 접두사 없이 씁니다.

How to Use This System

Usage
1
Reference in your prompt
Start any Claude prompt with: "Generate an HTML file using the design system at design-system.html. Copy its CSS variables block and use its component classes."
2
Use the /html slash command
The /html command in Claude Code automatically applies this design system. Try /html spec, /html review, or /html report followed by your content.
3
Copy the CSS variables block
The :root { ... } block at the top of this file's <style> tag is the single source of truth. Paste it into any new HTML file and reference the tokens throughout.
4
Stay self-contained
All generated files must be standalone HTML with no external CDN or stylesheet dependencies. Inline all CSS and JavaScript. This ensures files open correctly anywhere — browser, Claude artifact viewer, email.
5
Prefer SVG over ASCII for diagrams
Use inline <svg> for all charts, flow diagrams, and architecture sketches. SVG scales perfectly at any resolution and is fully styleable with CSS variables.
